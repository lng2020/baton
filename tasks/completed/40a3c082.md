# Add clear conversation button and auto-clear on plan confirm

Two changes needed:

**1. Add a 'Clear' button to the chat header in `frontend/index.html`**

In `frontend/index.html` around line 27-29, inside the `.chat-header-actions` div, add a clear button BEFORE the existing collapse toggle button:

```html
<div class="chat-header-actions">
    <button class="btn-chat-clear" id="btn-chat-clear" title="Clear conversation">&#10005;</button>
    <button class="btn-chat-toggle" id="btn-chat-toggle" title="Collapse chat">&#9660;</button>
</div>
```

**2. Style the clear button in `frontend/css/style.css`**

After the `.btn-chat-toggle:hover` rule (around line 589), add styling for the clear button that matches the toggle button style:

```css
.btn-chat-clear {
    background: transparent;
    border: none;
    color: var(--text-muted);

    cursor: pointer;
    padding: 0.2rem 0.4rem;

.btn-chat-clear:hover { color: var(--failed); }
```

**3. Wire up the clear button and auto-clear on plan confirm in `frontend/js/app.js`**

(a) Find where DOM elements are grabbed (search for `btn-chat-toggle` or similar `getElementById`/`querySelector` calls). Add:
```js
const btnChatClear = document.getElementById("btn-chat-clear");
```

(b) Find the chat event listeners section (around line 451-454 where `btnChatToggle.addEventListener` is). Add a click listener for the clear button:
```js
btnChatClear.addEventListener("click", resetChat);
```

(c) In the `confirmPlan()` function (around line 433-449), after the successful task creation (after `loadTasks()` on line 445), add a call to `resetChat()` to clear the entire conversation:

Change:
```js
    chatPlanEl.style.display = "none";
    currentPlan = null;
    loadTasks();
```
To:
```js
    resetChat();
    loadTasks();
```

This replaces the manual `chatPlanEl.style.display = "none"` and `currentPlan = null` since `resetChat()` already handles both of those (it sets `currentPlan = null` and hides the plan element).
