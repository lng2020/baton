# Add image upload UI to task mode

type: feature

Add image attachment capability to the task creation form:

1. **Update HTML** (`frontend/index.html`):
   - Add a file input and drop zone to the task form (inside `.task-form-body`, between the textarea and submit button, around line 69-70):
   ```html
   <div class="task-attachments-area">
       <input type="file" id="task-image-input" accept="image/*" multiple style="display:none" />
       <button type="button" class="btn-attach-task" id="btn-task-attach">Attach Images</button>
       <div class="task-attachments" id="task-attachments"></div>
   </div>
   ```

2. **Update JavaScript** (`frontend/js/app.js`):
   - Add DOM refs for `task-image-input`, `btn-task-attach`, `task-attachments`
   - Add state: `let taskPendingImages = []`
   - Wire `btn-task-attach` click to trigger `task-image-input.click()`
   - On `task-image-input` change: create preview thumbnails with remove buttons in `#task-attachments`
   - Modify `submitTask()` function:
     a. Upload each pending image via `POST /api/projects/{projectId}/upload` using `FormData`
     b. Append image references to the task content (e.g., `\n\n## Attached Images\n![image](/uploads/filename.png)` for each)
     c. Clear `taskPendingImages` and thumbnails after successful submission

3. **Update CSS** (`frontend/css/style.css`):
   - Style `.btn-attach-task` as a dashed-border area or button matching the dark theme
   - Style `.task-attachments` as a flex row of thumbnails similar to chat attachments
   - Style `.task-attachments-area` with appropriate spacing

4. **Update task detail rendering** in `renderDetail()` function (`frontend/js/app.js`):
   - When rendering task content, detect markdown image syntax `![...](/uploads/...)` and render as actual `<img>` tags
   - Or render the full content section using a simple markdown-to-HTML converter for images
